# Setup Guide

## Prerequisites

- [Bun](https://bun.sh) installed
- [Docker](https://docker.com) installed and running

## Database

### Overview

The project uses **PostgreSQL** (running in Docker) with **Drizzle ORM** for schema definition and migrations.

- Docker service: `saas-postgres` (defined in `docker-compose.yml`)
- ORM: Drizzle (schema in `db/schema.ts`, migrations in `migrations/`)
- Connection: via `pg.Pool` in `lib/drizzle.ts`
- Config: `drizzle.config.ts` for Drizzle CLI (migrate, push, studio)

### Environment Variables

Create `.env` in the project root (or copy from `.env.example`):

```env
POSTGRES_USER=user
POSTGRES_PASSWORD=password
POSTGRES_URL=postgresql://user:password@localhost:5432/dev_db
```

| Variable            | Description                                                |
| ------------------- | ---------------------------------------------------------- |
| `POSTGRES_USER`     | PostgreSQL username (used by Docker and connection string) |
| `POSTGRES_PASSWORD` | PostgreSQL password                                        |
| `POSTGRES_URL`      | Full connection string for Drizzle and the app             |

### Start the Database

```bash
# Start PostgreSQL container (builds custom image if needed)
docker compose up -d postgres

# Verify it's running
docker compose ps

# Stop when done
docker compose down

# Stop and delete all data
docker compose down -v
```

### Install Packages

Packages are already in `package.json`. If starting fresh on a new machine:

```bash
bun install
bun add drizzle-orm pg
bun add -d drizzle-kit
```

### Run Migrations

Migrations are SQL files generated by Drizzle Kit and stored in `migrations/`.

```bash
# Apply all pending migrations
bun run db:migrate
```

The project currently has 2 migrations:

| File                        | What it creates                                                                |
| --------------------------- | ------------------------------------------------------------------------------ |
| `0000_init_better_auth.sql` | Core tables: `user`, `session`, `account`, `verification`                      |
| `0001_glossy_quasar.sql`    | Plugin tables: `passkey`, `two_factor` + admin/2FA columns on `user`/`session` |

### Schema

6 tables defined in `db/schema.ts`:

| Table          | Purpose                                                |
| -------------- | ------------------------------------------------------ |
| `user`         | User accounts (name, email, role, banned, 2FA enabled) |
| `session`      | Active login sessions (token, expiry, IP, user agent)  |
| `account`      | Auth accounts (email/password hash, OAuth tokens)      |
| `verification` | Email verification and password reset tokens           |
| `passkey`      | WebAuthn/passkey credentials                           |
| `two_factor`   | TOTP secrets and backup codes                          |

### Adding New Better Auth Plugins

If you add a new Better Auth plugin, regenerate the schema and create a migration:

```bash
# Generate schema from auth config
npx @better-auth/cli generate --config ./lib/auth.ts --output ./auth-generated/better-auth-schema.ts

# Merge into db/schema.ts
# import: export * from "../auth-generated/better-auth-schema"

# Generate migration from schema changes
bunx drizzle-kit generate

# Apply
bun run db:migrate
```

### Useful Commands

```bash
# Open Drizzle Studio (web UI to browse data)
bun run db:studio

# Push schema changes directly (without generating migration files)
bun run db:push

# Generate a new migration from schema changes
bunx drizzle-kit generate

# Clear all data from all tables (keeps tables)
bun run db:clear
```

### Connection Details

The app connects to PostgreSQL in two places:

1. **`lib/drizzle.ts`** — runtime connection via `pg.Pool` using `env.POSTGRES_URL`
2. **`drizzle.config.ts`** — Drizzle CLI uses `process.env.POSTGRES_URL` for migrations/studio/push

Both use the same `POSTGRES_URL` from `.env`.
